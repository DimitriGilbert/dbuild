So here's the thing about using AI heavily as a developer — you burn through rate limits like there's no tomorrow. And every provider has a different way of showing you how much you've got left.

Codex? You gotta dig into their backend API. OpenRouter? Different endpoint, different format. Z.ai? Yet another API with yet another response structure. And none of them have a simple "hey, how much time do I have left?" command.

I got tired of juggling browser tabs and API documentation, so I did what any reasonable developer would do: I wrote a bash script about it.

Meet **LimitAID**. Because apparently even my AI tools need first aid when I'm done with them.

## What does it do?

LimitAID is a bash CLI that queries your AI providers and shows you quota, balance, and usage — all from one place, in your terminal, with pretty colors and progress bars.

```bash
limitaid all
```

That's it. One command. Three providers. Everything you need to know.

## The providers

### Codex / OpenAI (+ Spark)

```bash
limitaid codex
```

Shows your plan type, primary window usage (usually 5h), secondary window (weekly), credits balance, and Spark limits if you've got them. Auto-discovers your token from `~/.codex/auth.json`, so you don't even need to configure anything.

### OpenRouter

```bash
limitaid openrouter
```

Key balance, remaining credits, daily/weekly/monthly usage, free-tier status. Supports multiple keys — because of course you have both a work key and a personal key (don't lie, we all do).

### Z.ai

```bash
limitaid zai
```

Plan level, 5h coding window usage, weekly token limits, and — this is the cool part — a per-MCP breakdown of your tool usage. Want to know how much you spent on search-prime vs web-reader vs zread? Now you can. Auto-discovers your key from `~/.config/opencode/opencode.json`.

### All of them at once

```bash
limitaid all
```

Runs all three providers sequentially. Boom. Full picture.

## Loop mode

This one I use *constantly*. When I'm deep in an AI coding session and I need to keep an eye on my limits without switching away from what I'm doing:

```bash
limitaid all --loop 60
```

Clears the terminal and redraws every 60 seconds (minimum 30s, I'm not a monster). A dashboard for your AI quota, right there in your terminal. It's like having a fuel gauge for your coding speed.

## JSON output

For when you want to pipe things into other things (we all love piping things into other things):

```bash

```

Returns the raw provider response payloads. Feed it to `jq`, pipe it into a script, build your own dashboard, whatever floats your boat.

## Configuration

LimitAID supports three ways to feed it API keys, in priority order:

**1. CLI override** (highest priority):

```bash
limitaid zai --provider-config 'zai::your-api-key::personal'
```

**2. Config file** (`~/.config/limitaid/keys.conf`):

```ini
openrouter_work=/home/user/.secrets/openrouter_work
openrouter_personal=/home/user/.secrets/openrouter_personal
zai_personal=/home/user/.secrets/zai_personal
```

**3. Auto-discovery** (fallback):

Codex reads from `~/.codex/auth.json`, Z.ai reads from `~/.config/opencode/opencode.json`. OpenRouter doesn't auto-discover because, well, where would it even look?

## Requirements

Three things. That's it.

* `bash` (4.3+ for associative arrays)
* `curl` (API calls)
* `jq` (JSON parsing)

No Node.js. No Python. No npm install. No package manager drama. Just bash and two utilities that are probably already on your system.

## Why bash?

I know, I know. "But TypeScript is so much better!" "But Go compiles to a single binary!" "But Rust is fast!"

Yes. And I write all of those too. But for something that I want to install in 2 seconds, run everywhere, and not have to think about runtime versions or compilation targets? Bash hits different.

Plus, [parseArger](/blog/Projects/parseArger) generates all the argument parsing and completion for me, so the CLI plumbing was essentially free. The actual provider logic is maybe 100 lines per service. The rest is just formatting output to look nice in a terminal.

Sometimes the simple tool is the right tool.

## The name

LimitAID. Like "first aid" but for your AI limits. Because when you're in the zone and suddenly hit a rate limit, it feels like a medical emergency. At least it does for me ^^'

## Project structure

```text
LimitAID/
├── limitaid          Main entrypoint
├── bin/codex         Codex/OpenAI + Spark
├── bin/openrouter    OpenRouter balance
├── bin/zai           Z.ai coding plan
├── bin/all           All providers
├── lib/config.sh     Key loading + auto-discovery
├── lib/http.sh       HTTP helpers
├── lib/time.sh       Time formatting
├── lib/output.sh     Pretty output helpers
```

Clean, small, easy to understand. Each provider is its own script under `bin/`, shared logic lives in `lib/`. The main `limitaid` entrypoint just routes to the right subcommand.

So, who here uses **git**? And I mean *really* uses it, not just `commit -m "stuff"` and push it when things break?

Yeah, git is great and all, but sometimes you look at `git log` and it's like trying to read the matrix code upside down. There's so much data there. So many patterns. So many... questions! Like "who actually broke production last month?" or "is this repo a one-person show?" or "why does this file have more commits than my entire project history?"

Well, I got tired of staring at commit hashes, so I built **git-moar**! üòõ

## What is git-moar?

[git-moar](https://github.com/DimitriGilbert/git-report) is a CLI tool that takes your git repository (or repositories, if you're feeling adventurous) and turns it into a beautiful, interactive HTML report with charts, metrics, and insights that might just make you question your life choices (in a good way... I think).

Two main commands:
* **`snitch`** ‚Äî analyze a single repository (great for stalking... I mean, understanding... a particular project)
* **`scattered`** ‚Äî analyze multiple repositories at once (perfect for that "oh god, what have we created" moment)

Think of it as git log, but with actual colors, charts, and the kind of data you can show to management without them falling asleep. Not state of the art, but sure beats `grep`! üòä

## Quick start

```bash
npm install -g git-moar

cd my-project
git-moar snitch

# or analyze multiple repos at once (wildcards work, 'cause magic)
git-moar scattered ~/projects/*/

open git-moar-report.html  # or xdg-open on Linux, or start on Windows
```

## What's in the report?

Ah, the juicy part! 15+ chart types and more metrics than you can shake a branch at.

### Quality metrics (because we all pretend to care)

* **Bus Factor** ‚Äî how many people need to get hit by a bus before your project dies? Spoiler: probably 1 for most of us
* **Code Churn** ‚Äî lines added vs deleted, with a "who's actually deleting code" breakdown
* **Health Score** ‚Äî a number you can use to feel good or terrible about yourself
* **Gini Coefficient** ‚Äî inequality in contributions, because economics can ruin anything

* **Velocity** ‚Äî commits over time, with trends and the inevitable "what happened in December" dip
* **Work-Life Balance** ‚Äî weekend vs weekday commits (prepare to be judged)

### File hotspots (the drama queens of your repo)

* **Frequently Changed Files** ‚Äî files that get touched more than your phone
* **Knowledge Silos** ‚Äî files that only one person ever touches (recipe for disaster)

### Other cool stuff

* **Commit Classification** ‚Äî are you writing features, fixes, docs, or just "update"?
* **Message Quality Scoring** ‚Äî how terrible are your commit messages? (spoiler: probably very)
* **GitHub-Style Contribution Calendar** ‚Äî show off your green squares (or lack thereof)
* **Sortable Data Tables** ‚Äî filter and sort contributors, files, commits, whatever

## Visualizations

The report uses [Chart.js 4.4.0](https://www.chartjs.org/) for all the pretty charts, and let me tell you, I was pretty proud when I got the heatmaps working. Here's what you get:

* Activity trends over time

### Export options

Sometimes you need data for spreadsheets (or that executive who demands CSV files for some reason). git-moar can export to:

* **CSV** ‚Äî because Excel still rules the corporate world apparently
* **JSON** ‚Äî for when you want to parse things programmatically or feed it into some AI that tells you your repo is doomed

### Multi-repo analysis

The `scattered` command is where things get interesting. Analyze multiple repositories and get cross-project insights. Want to know which repo gets the most commits across your whole org?
